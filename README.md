# כובש ההר - תיעוד והפעלה

פלטפורמה חינמית לשיתוף סיפורי רכיבה (אינדורו כביש, סינגלים, הארד אינדורו).

## מבנה הפרויקט

```
roadstory/
  backend/     # FastAPI + Postgres
  frontend/    # Next.js + Tailwind
```

## הרצה מקומית

### Backend
```bash
cd backend
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```
בלי משתני סביבה זה ירוץ על SQLite מקומי (`local_dev.db`) - טוב לפיתוח, לא לפרודקשן.

### Frontend
```bash
cd frontend
npm install
npm run dev
```
פתח `.env.local` עם:
```
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## פריסה לפרודקשן (בהתאם לתשתית שכבר יש לך)

### 1. Backend → Railway
- צור שירות חדש מה-repo, root directory: `backend`
- הוסף Postgres plugin - Railway יזריק `DATABASE_URL` אוטומטית
- Start command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
- משתני סביבה נדרשים:
  - `JWT_SECRET_KEY` - מחרוזת אקראית ארוכה
  - `ADMIN_EMAILS` - כתובות אימייל של מנהלים, מופרדות בפסיקים (למשל `or@example.com`) - מאפשר להם למחוק כל סיפור, לא רק את שלהם
  - `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME`, `R2_PUBLIC_URL_BASE`

### 2. אחסון קבצים - שתי אופציות

**אופציה א׳ (ברירת מחדל, בלי אשראי/הרשמה נוספת): Volume על Railway עצמו**
- בשירות ה-backend ב-Railway: **Settings** → **Volumes** → **+ New Volume**
- Mount path: `/data`
- זהו - אין צורך במשתני סביבה נוספים, `STORAGE_BACKEND` ברירת המחדל שלו הוא `local`
- חשוב: תוסיף משתנה `PUBLIC_BASE_URL` עם כתובת ה-backend שלך ב-Railway (למשל `https://sipur-shvil-production.up.railway.app`, **בלי** `/` בסוף) - כדי שהקישורים לתמונות ייבנו נכון

**אופציה ב׳ (לעתיד, כשנוח לך עם הרשמה+אשראי): Cloudflare R2**
- יתרון: CDN מובנה בלי עמלת יציאה, מתאים יותר כשהתנועה גדלה
- צור bucket חדש, חבר אליו custom domain ציבורי (זה ה-`R2_PUBLIC_URL_BASE`)
- צור API token עם הרשאות Object Read & Write
- הוסף את משתנה הסביבה `STORAGE_BACKEND=r2` + חמשת משתני ה-R2 (ראה `.env.example` בתיקיית backend)
- **המעבר בין השניים הוא שינוי משתנה סביבה אחד - שום קוד לא צריך להשתנות**

### 3. Frontend → Vercel
- חבר את ה-repo, root directory: `frontend`
- משתנה סביבה: `NEXT_PUBLIC_API_URL` = כתובת ה-Railway של ה-backend
- Vercel פרוס אוטומטית עם כל push ל-main

### 4. מפות → OpenFreeMap (בלי שום הרשמה, מפתח, או אשראי)
- אין שום דבר לעשות - זה כבר מוגדר בקוד ועובד מהקופסה
- מבוסס OpenStreetMap, ללא הגבלת שימוש, קוד פתוח (MapLibre GL)
- (Mapbox היה חלופה אפשרית, אבל דורש כרטיס אשראי בהרשמה - הוחלף בכוונה)

## מגבלות בכוונה (לשמירה על עלות אפסית בגדילה)
- עד 10 תמונות לסיפור, עד 8MB לתמונה
- קובץ GPX עד 5MB
- וידאו לא מאוחסן באתר - בשלב זה מומלץ לקשר ליוטיוב/אינסטגרם בגוף הסיפור

## איך זה בנוי כדי לעבוד גם בסקייל גדול

**מיקומים - הבסיס לחיפוש:** במקום שדה טקסט חופשי, כל סיפור מסווג לפי `country` + `region`
(לישראל - רשימה סגורה של 12 אזורי רכיבה; לחו"ל - מדינה מרשימה + שם מקום חופשי). שני
השדות מאונדקסים ב-DB, כך שסינון לפי אזור נשאר מהיר גם עם עשרות אלפי סיפורים.

**אינדקסים שכבר קיימים:** `ride_type`, `difficulty`, `country`, `region`, `created_at`,
ו-`is_published` - כל הפילטרים הנפוצים בפיד. מיון כרונולוגי (הכי נפוץ) מהיר תודות
לאינדקס על `created_at`.

**עימוד:** ה-API מגביל `limit` ל-50 מקסימום בכל בקשה, כך שאי אפשר "לשבור" את השרת עם
בקשה לכל הסיפורים בבת אחת.

**אחסון מדיה:** Cloudflare R2 לא גובה עמלת יציאה (egress) - זה ההבדל המהותי מול S3
כשכמות התעבורה גדלה. בנוסף יש מגבלות עיצוביות (עד 10 תמונות/8MB לתמונה, GPX עד 5MB)
שמונעות התנפחות עלות.

**מה שכדאי לשדרג כשהתנועה תגדל משמעותית (לא צריך את זה ביום הראשון):**
- **ספירת לייקים/תגובות** - כרגע מחושבת בכל בקשה (`COUNT` מה-DB). ב-~10,000+ סיפורים
  כדאי לעבור לעמודת מונה מתעדכנת (`like_count` על הטבלה עצמה, מתעדכנת ב-trigger או
  באפליקציה) כדי לחסוך שאילתת COUNT נפרדת לכל סיפור בפיד.
- **חיפוש טקסט חופשי** - כרגע `ILIKE` פשוט על הכותרת. זה מספיק לאלפי סיפורים, אבל
  לא סקיילבילי לעשרות אלפים. השדרוג: Postgres Full-Text Search (עמודת `tsvector` +
  אינדקס GIN), ואם צריך יותר - מנוע חיפוש ייעודי כמו Meilisearch או Typesense.
  Meilisearch לוקח 10 דקות להרים ב-Railway.
- **עיבוד GPX ותמונות** - כרגע קורה בתוך הבקשה עצמה (סינכרוני). כשההעלאות יהיו
  תכופות/כבדות, כדאי להעביר את זה ל-job queue (Celery/RQ + Redis) כדי שההעלאה תחזיר
  תשובה מיידית והעיבוד יקרה ברקע.
- **קאשינג** - פיד הבית והסיפורים הפופולריים הם מועמדים טובים ל-Redis cache
  (רענון כל כמה דקות), כדי להוריד עומס מה-DB בעומסי קריאה גבוהים.
- **Connection pooling** - כש-Railway Postgres מתחיל לקבל הרבה חיבורים במקביל,
  PgBouncer מונע "אזל בחיבורים" לפני שצריך לשדרג את מסד הנתונים עצמו.

**אבטחה - דבר אחד לשדרג לפני production אמיתי:** הטוקן (JWT) כרגע נשמר ב-`localStorage`
בפרונט - פשוט ועובד ל-MVP, אבל חשוף יותר ל-XSS מאשר cookie מסוג `httpOnly`. לפני
שיש כאן מידע רגיש או משתמשים בהיקף גדול, כדאי לעבור לניהול session מבוסס cookies.

## מה בנוי ומה נשאר

**בנוי ועובד מקצה לקצה:**
- מודל DB מלא: משתמשים, סיפורים, תמונות, לייקים, תגובות, עדכוני שטח - עם אינדקסים לסקייל
- הרשמה/התחברות עם JWT
- העלאת סיפור מלאה: טקסט + GPX (מחושב אוטומטית מרחק/טיפוס/פרופיל) + עד 10 תמונות
- **סיווג דו-ממדי**: סוג אופנוע (אדוונצ'ר/דו-שימושי/אנדורו/מוטוקרוס) + סגנון רכיבה (טיול/מהיר/טכני/הארד אנדורו) בנפרד, כמו במסמך האפיון
- **עונה מומלצת** (חורף/קיץ/כל השנה) כשדה וכפילטר
- **נקודת כינוס + ניווט ישיר** - שדה ידני (עם כפתור "המיקום שלי") שיוצר כפתורי Waze/Google Maps ישירות בעמוד הסיפור
- **עדכוני שטח** - כל רוכב יכול לדווח על מצב המסלול (פתוח/חסום/בוצי) עם הערה קצרה ותאריך
- מיקום קטגורי: מדינה + אזור (ישראל: 12 אזורים מוגדרים; חו"ל: מדינה + מקום חופשי)
- תגובות ולייקים - API + ממשק מלא
- **הצהרת אחריות משפטית** - חובה לאשר צ'קבוקס בהרשמה (מתועד עם timestamp ב-DB), ומוצגת גם בפוטר קבוע בכל עמוד
- **מניעת ספאם** - מינימום 30 מילים בתיבת הסיפור (נאכף גם בשרת וגם בטופס), מונע פוסטים ריקים/בדיקה
- **מחיקת סיפורים** - כל משתמש יכול למחוק את הסיפור שלו; משתמשים ב-`ADMIN_EMAILS` יכולים למחוק כל סיפור (מודרציה בסיסית בלי פאנל ניהול נפרד)
- **מפת לוויין + מסך מלא** - toggle בין מפה רגילה ללוויין (Esri, חינמי), וכפתור הגדלה למסך מלא
- **צור קשר** (`/contact`) - טופס פתוח לכל פנייה, נשמר ב-DB ונגיש למנהלים בעמוד `/admin`
- **תצוגה היברידית מפה+פיד** - מפת סקירה עם נעץ לכל סיפור, בעמוד הבית ובעמוד העיון
- **מקומות שמורים לפרסומות** (placeholder מעוצב) - בראש הפיד ובאמצעו, מוכן לחיבור רשת פרסום אמיתית בעתיד
- עמוד עיון (`/stories`) עם סרגל סינון מלא: מדינה→אזור, סוג אופנוע, סגנון רכיבה, קושי, עונה, חיפוש, ועימוד
- טופס העלאת סיפור מלא (`/stories/new`) כולל שער הרשמה/התחברות מובנה
- עמוד סיפור בודד (`/stories/[id]`) עם מפת מסלול, כפתור ניווט, עדכוני שטח, גלריה, לייקים ותגובות

**נשאר לבנות - "גיים־צ'יינג'רים" מהמסמך (דורשים תשתית נוספת, לא רק קוד):**
- **מצב בוץ בזמן אמת** - אפשר לבנות על גבי מערכת עדכוני השטח הקיימת, אבל דורש מחשבה על "פקיעת תוקף" של תיוג (עדכון מלפני חודש לא אמין)
- **חיבור לאפליקציית Off-Road** - דורש לבדוק אם יש להם API ציבורי; אם לא, אפשר להתחיל מ-QR code שמוריד את קובץ ה-GPX ישירות לנייד
- **כפתור חילוץ חירום** - הכי משמעותי אבל גם הכי מורכב: דורש התראות בזמן אמת (WebSocket/Push notifications) ומיקום live, וכדאי לחשוב היטב על אחריות משפטית לפני שבונים את זה (אתר לא-מקצועי שמתיימר לתאם חילוץ יכול ליצור ציפייה מסוכנת אם התגובה לא מגיעה)

**נשאר לבנות - "Nice to have":**
- עמוד פרופיל משתמש, לוח "מחפש שותף", דירוג קהילתי (Upvote/Downvote), עריכה/מחיקת סיפור, אימות עם Google
